Var vs Let vs Const

Además de var, ahora hay acceso a dos nuevos identificadoes para almacenar valores, estos son: let y const.
A diferencia de las declaraciones de var con las de let y const estas no permiten hoisted, lo que significa que las declaraiones no serán vistas fuera del alcance en el cual fueron asignados.

Ejemplo usando var:

var snack = 'Meow Mix';

function getFood(food) {
    if (food) {
        var snack = 'Friskies';
        return snack;
    }
    return snack;
}

getFood(false); // undefined

Ahora, si usamos let en vez de var el resultado va variar, como esto: 

let snack = 'Meow Mix';

function getFood(food) {
    if (food) {
        let snack = 'Friskies';
        return snack;
    }
    return snack;
}

getFood(false); // 'Meow Mix'

Este cambio en el comportamiento destaca que tenemos que tener cuidado al refactorizar el código heredado que utiliza var. Ciegamente reemplazar instancias de var con let puede conducir a un comportamiento inesperado.

Nota: let y const tienen un ámbito de bloque. Por lo tanto, hacer referencia a identificadores de ámbito de bloque antes de que se definan producirá un ReferenceError.
console.log(x); // ReferenceError: x is not defined

let x = 'hi';
Buena práctica: Dejar las declaraciones var dentro del código heredado para indicar que necesita ser refactorizado cuidadosamente. Cuando se trabaja en una nueva base de código, utilice let para variables que cambiarán su valor en el tiempo y const para variables que no se pueden reasignar.
