Modules

Antes de ES6, se utilizaba bibliotecas como Browserify para crear módulos en el lado del cliente, y requieren de Node.js. Con ES6, ahora se puede utilizar directamente módulos de todos los tipos como: AMD y CommonJS.

Exporting in CommonJS

module.exports = 1;
module.exports = { foo: 'bar' };
module.exports = ['foo', 'bar'];
module.exports = function bar () {};

Exporting in ES6

Con ES6, se tiene varias formas de exportación. Se puede realizar Named Exports:

export let name = 'David';
export let age  = 25;

Además se puede exportar una lista de objetos:

function sumTwo(a, b) {
    return a + b;
}

function sumThree(a, b, c) {
    return a + b + c;
}

export { sumTwo, sumThree };

También se puede exportar funciones, objetos y valores (etc.) simplemente usando la palabra clave export: 

export function sumTwo(a, b) {
    return a + b;
}

export function sumThree(a, b, c) {
    return a + b + c;
}

Y por último, se puede exportar enlaces predeterminados: 

function sumTwo(a, b) {
    return a + b;
}

function sumThree(a, b, c) {
    return a + b + c;
}

let api = {
    sumTwo,
    sumThree
};

export default api;

/* Which is the same as
 * export { api as default };
 */

Buena práctica: Utilizar siempre el método predeterminado export default al final del módulo. Esto deja en claro que es lo que se está exportando y ahorra tiempo al tener que averiguar qué nombre exprtó un valor. Más aún, la práctica común en los módulos de CommonJS es exportar un solo valor u objeto. Siguiendo este paradigma, hacemos que nuestro código sea fácilmente legible y nos permitamos interpolar entre los módulos CommonJS y ES6.

Importing in ES6

ES6 proporciona diversas maneras de importación. Podemos importar un archivo completo:

import 'underscore';
Es importante tener en cuenta que la simple importación de un archivo completo ejecutará todo el código en el nivel superior de ese archivo.

Similar a Python, se realizan las llamadas a las importaciones:

import { sumTwo, sumThree } from 'math/addition';

También se puede cambiar el nombre de las importaciones como se indican: 

import {
    sumTwo as addTwoNumbers,
    sumThree as sumThreeNumbers
} from 'math/addition';

Además, podemos importar todas las cosas (también llamadas namespace import): 

import * as util from 'math/addition';

Por último, se puede importar una lista de valores de un módulo: 

import * as additionUtil from 'math/addition';
const { sumTwo, sumThree } = additionUtil;

Ejemplo de un importación desde el enlace predeterminado como:

import api from 'math/addition';
// Same as: import { default as api } from 'math/addition';

Si bien es mejor mantener las exportaciones simples, pero a veces podemos mezclar la importación por defecto y la importación mixta si es necesario. Cuando estamos exportando así por ejemplo:

// foos.js
export { foo as default, foo1, foo2 };

Se puede también importar como los siguientes:

import foo, { foo1, foo2 } from 'foos';

Al importar un módulo exportado usando la sintaxis commonjs (como React) se puede hacer:

import React from 'react';
const { Component, PropTypes } = React;

Esto también se puede simplificar más, usando:

import React, { Component, PropTypes } from 'react';

Nota: Los valores que se exportan son enlaces, no referencias. Por lo tanto, cambiar el enlace de una variable en un módulo afectará al valor dentro del módulo exportado. Evite cambiar la interfaz pública de estos valores exportados.
